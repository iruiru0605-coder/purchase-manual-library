# 運用手順

## 購入履歴を取り込む

### CSVがある場合

1. `購入履歴Inbox` を開く
2. 購入元を選ぶ
3. CSVを選択する
4. `取り込む` を押す

Amazon、楽天、メルカリの正式CSV形式が違っても、列名から可能な範囲で自動判定します。

### CSVがない場合

Amazonの公式データリクエストは時間がかかる場合があります。普段は購入履歴ページから直接コピーします。

1. Amazonまたは楽天の購入履歴ページを開く
2. 注文カード周辺を範囲選択してコピーする
3. アプリの `購入履歴ページを貼り付け` に貼る
4. `貼り付けから取り込む` を押す

この方法は完全な公式エクスポートではないため、余計な行が候補に混ざることがあります。候補レビューで `不要` にしてください。

## 登録候補を仕分ける

候補ごとに次を選びます。

* `登録する`
* `不要`
* `後で確認`

食品や消耗品などは低優先、家電・家具・工具・子ども用品・健康器具・車用品などは高優先に寄せます。

## AI補完

`AI補完` を押すと、次を補完します。

* メーカー
* 商品名
* 型番
* カテゴリ
* 検索クエリ

LLM APIキー未設定時は、CSV列と商品名から簡易推定します。

## PDF候補探索

`候補を探す` を押すと、公式サイト優先でPDFを探します。

PDF候補は自動選択されますが、プレスリリースなど説明書ではないPDFは低スコアに落とします。登録前にチェックを外せます。

## 登録する

`選択したPDFを保存して登録` を押します。

Google Driveにログイン済みならDriveへ保存します。未ログインならローカルの `.manual-library/archive/` へ保存します。

## 紙マニュアル

登録時に次の形式で保管先を自動作成します。

```text
カテゴリ BOX-番号 #連番
```

例:

```text
キッチン家電 BOX-01 #001
工具 BOX-06 #003
```
